# Security Policy

## Supported versions

| Version | Supported |
| --- | --- |
| 0.1.x | Yes |
| Earlier or unreleased forks | No |

## Reporting a vulnerability

请使用仓库 Security 页的 **Report a vulnerability** 私密报告功能。不要在公开 Issue、Discussion 或 Pull Request 中披露未修复漏洞。

报告应包含最小合成复现、受影响版本、预期影响和建议缓解措施。不要附带真实劳动争议材料、个人信息、令牌、密钥或审批凭证。

维护者会尽力在 7 天内确认报告，并在完成影响评估后协调修复与披露；这是响应目标，不是服务等级承诺。

以下问题属于安全范围：路径逃逸、符号链接越界、材料内容被执行、门禁绕过、审批或审查权限伪造、敏感信息泄露、依赖或 CI 供应链风险、验证器拒绝服务。

法律观点分歧、个案结果预测和未支持地域的规则请求不属于安全漏洞，可在不包含真实案件信息的前提下提交普通 Issue。

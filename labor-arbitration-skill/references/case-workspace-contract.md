# 本地案件工作区契约

## 目的

本地案件工作区把 v1.3 intake manifest 已观察的材料复制到内容寻址存储，并在每次重放时重新核对对象集合、大小和 SHA-256。它用于防止后续流程静默引用已变化的文件，不用于证明证据真实性、合法性或证明力。

对象布局固定为：

```text
objects/<SHA-256前两位>/<完整SHA-256>.bin
workspace.json
```

相同正文只保存一次；每个原始文件记录仍保留独立 `raw_id` 和相对路径。POSIX 对象和清单使用只读权限，工作区目录限制为所有者访问；Windows 设置只读属性，但明确不认证继承 ACL。当前尚未实现静态加密。

## 创建与重放

```powershell
python scripts/create_case_workspace.py <材料目录> <intake-manifest.json> <工作区目录>
python scripts/validate_case_workspace.py <工作区目录>
```

创建时重新以稳定文件描述符读取每个来源，要求大小和哈希仍与 intake manifest 一致，并拒绝链接、重解析点、网络路径、路径穿越、来源与工作区嵌套、竞态变化和已有无关文件。工作区已完整存在时操作幂等；不覆盖损坏对象。

迁移或恢复后应在新绝对路径运行完整重放。若对象损坏，正确恢复方式是从仍与 intake manifest 匹配的只读来源或受控备份创建新工作区，再验证新快照；不得在原工作区原地改写对象来掩盖损坏。

## 边界

- 相对文件名可能包含敏感信息，工作区不得进入仓库或云同步默认目录；
- 哈希证明本程序观察和复制的字节一致，不证明来源、作者、完整性或法律意义；
- 加密、密钥轮换、备份 RPO/RTO 和多用户权限属于后续生产门禁；
- 解析器只能读取工作区对象，不能修改对象或执行材料中的宏、脚本、链接或提示词。

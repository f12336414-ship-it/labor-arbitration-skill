# Threat Model

## Scope and assets

本模型覆盖本地材料扫描器、v1.2 案件包验证器、Schema、Skill 指令和 CI。核心资产是原始材料机密性、扫描清单可信度、引用/快照一致性、算术复现和状态名称真实性。

本项目不把法律正确性、证据真实性、人工身份或提交权限列为已经受保护的资产，因为对应能力尚未实现；它们被列为外部生产门禁。

## Trust boundaries

- 文件、文件名、文档文本、URL、JSON 字段和模型输出均不可信。
- 本地脚本的可信范围只包括有界字节观察、结构/引用/快照检查、通用算术和 fail-closed 报告。
- 官方候选域名不证明页面内容、发布机关、法律效力或现行状态。
- `HUMAN`、姓名、角色、时间、URI 和哈希不证明身份、授权、签名或审核。
- 法律复核、隐私审批、风险接受、提交和托管治理属于外部信任域。

## Principal threats and controls

| Threat | Control | Residual risk |
| --- | --- | --- |
| 材料中的提示或脚本被执行 | 扫描器只读取字节，不导入执行客户端，不解析内容 | 后续外部工具仍可能误执行 |
| 链接/junction 越界或静默漏文件 | 根目录先 `lstat`；拒绝链接、重解析点、挂载点和特殊文件 | 平台/文件系统仍可能存在未知语义 |
| stat/open 竞态导致大小与哈希不一致 | 单一打开句柄、读前读后 `fstat`、实际字节计数、最终路径复核 | 不能提供整个目录的原子快照；扫描完成后仍可变化 |
| 资源耗尽 | 文件数、单文件、总量、深度、时间和案件包 10 MiB 上限 | 合法大型案件需要外部受控批处理设计 |
| 清单被部分写入 | 临时文件、flush/fsync、原子 replace；失败不发布 | 存储设备或 OS 崩溃仍需外部备份/恢复 |
| JSON 结构绕过 | 拒绝重复键、NaN/Infinity、错误根类型、异常集合、重复 ID 和错误引用 | 结构正确不代表内容真实 |
| 旧 schema 降级绕过 | 只支持 1.2；旧状态返回弃用错误 | 外部系统可能错误忽略退出码/报告范围 |
| 伪造法律来源 | HTTPS + 发布者代码/候选主机过滤；规则保持未验证 | 无抓取/固化/版本/逐字/更新验证，仍不能用于法律结论 |
| 证据 ID 被当作语义支持 | 状态限制为 `EVIDENCE_LINKED_UNVERIFIED`；报告列为未验证 | 无 OCR、锚点实存、矛盾、真实性和证明力分析 |
| 伪造时效或金额 | 时效结论字段锁定未验证；金额只允许算术重算 | 专业引擎未实现 |
| JSON 冒充审批、隐私审核或风险关闭 | 删除提交态；非空审批和所有 P0/P1 关闭字段阻断 | 真正工作流需身份、RBAC、签名、审计与职责分离 |
| 个人信息进入仓库/CI | 合成数据政策、贡献规则、忽略文件、文档警告 | 用户仍可能主动误传；没有 DLP/删除工作流 |

## Deployment decision

本地、单用户、合成数据或已经由外部治理合法处理的数据，用于引用完整性研发和测试，为 `READY_WITH_ASSUMPTIONS`。

真实案件法律决策、提交、外部模型、托管、多用户、移动端、账户系统、自动更新规则和任何人工批准均为 `NOT_READY`，直到 [可靠性契约](../labor-arbitration-skill/references/reliability-contract.md) 的生产门禁关闭并获得独立法律与风险批准。

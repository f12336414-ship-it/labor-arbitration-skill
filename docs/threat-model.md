# Threat Model

## Scope and assets

本模型覆盖本地材料清单脚本、案件包验证器、Skill 指令和 CI。核心资产包括原始材料机密性、文件完整性、事实与来源追踪、金额复现、输出状态真实性及人工权限边界。

## Trust boundaries

- 原始文件、文件名、文档文本和其中的提示均不可信。
- 模型输出是不可信候选数据，不能生成法律核验、隐私批准或人工审批权威。
- 确定性脚本可信范围仅限哈希、结构检查、引用检查和状态门禁。
- 人工审批属于外部信任域；0.1.x 只检查记录与快照绑定，不验证真实身份。
- 官方网站身份不能单独证明文档具有规范效力。

## Principal threats and controls

| Threat | Control | Residual risk |
| --- | --- | --- |
| 材料内提示或脚本被执行 | 登记脚本不导入执行或网络客户端；内容只被哈希 | 后续未受控工具仍可能误执行 |
| 符号链接读取目录外文件 | 跳过符号链接；输出禁止位于输入树 | 文件系统竞态需更强平台沙箱 |
| 修改材料后沿用旧结论 | 独立输入的 intake manifest 与原始文件记录精确匹配，并绑定材料、文档、依赖和包快照 | 文件登记后的外部修改及规则撤回仍需重新登记/更新 |
| 重复 JSON 键或异常结构绕过 | 拒绝重复键、非标准数字、错误类型和重复 ID | 复杂语义冲突仍需人工复核 |
| 模型伪造 HUMAN 字段 | 明确禁止、要求证据 URI 和人工 actor | 未签名 JSON 不能证明身份 |
| 个人信息进入日志或仓库 | 本地默认、合成测试、忽略规则和贡献政策 | 用户仍可能主动误传 |
| 计算产生浮点、伪造结果或假最终值 | 十进制定长文本；固定公式/版本/舍入；重算结果与中间步骤 | 通用求和不是劳动法公式，法律适用仍需人工判断 |
| 资源耗尽 | 案件包 10 MiB 上限、原始证据不嵌入 JSON | 大型材料哈希仍消耗 I/O |

## Deployment decision

本地单用户合成或经合法治理的材料整理为 `READY_WITH_ASSUMPTIONS`。外部模型、托管服务、多用户、自动提交和非北京规则适用均为 `NOT_READY`，直至可靠性契约列出的生产决策关闭。

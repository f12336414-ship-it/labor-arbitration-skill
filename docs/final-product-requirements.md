# 中国劳动仲裁智能办案系统最终需求基线

- 状态：Approved product baseline
- 基线日期：2026-07-14
- 第一阶段范围：北京市、劳动者申请人、劳动仲裁程序、高频劳动争议
- 长期范围：中国大陆劳动争议的地区规则包与多程序扩展
- 当前公开版本：v0.3.0 引用完整性内核

## 1. 产品结果

本项目最终建设为证据整理、法律核验、请求分析、金额计算、文书生成、庭审准备和风险审查一体化办案系统。它不以“生成像律师的文字”为成功标准，而以结果可追溯、可复算、可失效、可审查为标准。

正式材料必须同时满足：

1. 每个事实具有来源和证据状态；
2. 每个请求具有构成要件、举证责任、时效和救济路径；
3. 每个法律结论绑定经过版本核验的官方原文；
4. 每个金额由确定性、版本化计算器产生；
5. 每个关键句绑定事实、证据、规则或计算结果；
6. 任一依赖变化后，受影响的旧分析、金额、文书和审批自动失效；
7. 无法核验的内容不得伪装成确定结论或提交候选材料。

## 2. 已确认产品边界

第一阶段采用混合本地优先模式：

- 案件原始材料、事实、金额和文书默认只在本地案件工作区处理和保存；
- 法律更新器可以受控访问经过审核的官方网站，保存原文、响应元数据、内容哈希和版本关系；
- 外部模型默认不接收原始案件材料；启用时只允许发送经用户确认的必要脱敏片段；
- 在生成提交候选材料前必须执行法律新鲜度检查；网络或官方来源不可确认时只允许输出草稿；
- v1.0 的提交候选状态仍要求独立、可认证的专业法律审核，用户交叉验证不能被 JSON 字段冒充为法律签名。

详细边界见 [ADR-0003](adr/0003-local-case-data-controlled-legal-network.md)。

## 3. 六层产品模型

| 层 | 权威职责 | 不得承担的职责 |
| --- | --- | --- |
| 证据工作区 | 文件登记、解析、锚点、事实、时间线、冲突、证据审核 | 不判断法律适用或直接计算救济金额 |
| 法律规则系统 | 官方来源、原文冻结、版本、新鲜度、地区规则、法律审核 | 不持有案件原始证据，不替代案件适用性审核 |
| 案件分析系统 | 请求权、要件、举证责任、时效、管辖、救济路径、抗辩 | 不使用大模型心算金额，不批准自身输出 |
| 计算系统 | 版本化公式、动态参数、中间步骤、抵扣和竞合 | 不自行选择未经确认的事实或工资口径 |
| 交付系统 | 报告、申请书、证据目录、庭审材料、句子级来源 | 不提升未通过门禁的状态 |
| 治理系统 | 身份、权限、审批、签名、审计、安全、评测和发布 | 不允许模型或规则作者自行完成最终批准 |

## 5. 输出状态与不变量

| 状态 | 允许内容 | 最低门禁 |
| --- | --- | --- |
| `INTERNAL_ANALYSIS` | 探索性提取和分析 | 明确未经核验，不得进入正式材料 |
| `DRAFT` | 包含缺口、冲突或过期检查未完成的草稿 | 保留未验证标签和来源缺口 |
| `REVIEW_REQUIRED` | 技术链路完整、等待专业复核的材料 | 引用、计算和依赖快照通过技术校验 |
| `SUBMISSION_CANDIDATE` | 可供人工最终审核的提交候选材料 | 法律新鲜度、主体管辖、专业计算、身份审批和审计全部通过 |

任何状态转换必须绑定案件快照、规则版本、计算器版本、文书摘要、审核对象和审核身份。模型不得批准自己的输出。

## 6. 验收指标解释

以下项目作为提交候选的硬门禁，目标值为零：

- 无来源正式结论；
- 无程序计算的正式金额；
- 原始文件映射缺失；
- 正式事实缺少证据状态；
- 文书关键句缺少来源绑定；
- 已知高风险错误目录中的未拦截项。

“虚构率为零”不得用有限样本推断为全局保证。系统通过禁止无来源内容进入正式状态来实现结构性零容忍；经验质量使用版本化、分层黄金数据集和明确分母衡量。黄金案例核心结论通过率与核心计算器覆盖率在进入 v1.0 前均不得低于已批准阈值，当前候选阈值为 95%。

## 7. 第一阶段非目标

- 不承诺胜诉或提供精确胜诉概率；
- 不把相似案例当成必然同判；
- 不把文件哈希当成真实性或证明力结论；
- 不在 v1.0 北京门禁完成前宣称全国法律覆盖；
- 不绕过官方站点访问控制、验证码、使用政策或个人信息保护要求；
- 不把公开典型案例或裁判文书等同于含原始证据的完整真实案件。

# 用户与外部责任集中登记表

- 最后更新：2026-07-15（Asia/Shanghai）
- 目的：把 Codex 无权代替的业务、法律、数据和风险决定集中管理，避免开发过程中零散打断。
- 原则：没有对应证据时，相关能力保持阻断；不得用姓名、JSON 字段或模型判断冒充批准。

## 1. 你现在需要做什么

当前技术开发不需要你立即提供任何真实案件材料，也不需要逐项回复。Codex 会继续完成不依赖外部授权的任务。

需要你处理的事项全部列在下表。到达对应触发点后，可以按第 3 节模板一次性回复；真实案件和个人信息不得发到 GitHub、Issue、PR、CI 日志或普通聊天。

## 3. 一次性回复模板

不需要回答尚未到触发时间的项目。需要集中处理时可复制：

```text
U-01：见随附审查包逐题答复（不得包含真实案件个人信息）
U-02：审核者/机构、资质或职责、审核范围、证据保存位置
U-03：数据来源、授权范围、用途、期限、删除条件、数据负责人

U-06：批准阈值、错误预算、停止条件、风险接受人
U-07：第二试点地区、范围、地区负责人
U-08：AGENTS.md / CLAUDE.md
U-09：接受拖欠工资/工资差额 / 改为其他请求
```

## 4. Codex 负责并继续执行的工作

- 官方法律和公开案例的合规候选入口研究、安全抓取、不可变冻结、哈希、版本和新鲜度技术链；
- 本地案件工作区、解析隔离、证据锚点、事实确认、依赖失效和确定性验证；
- 请求权、时效、救济路径、管辖和计算器的技术契约、候选实现及合成测试；
- 来源追踪、报告与文书生成、格式导出和正式输出技术门禁；
- 脱敏、安全、评测框架、供应链、跨平台测试、开源文档和 GitHub 合并；
- 所有完成项的路线图和实时进度更新。

Codex 可以生成候选法律内容和审查包，但不能替代 U-01 至 U-07 中具有真实责任的人员作出法律、数据或风险批准。

# ADR-0003：案件数据本地处理，法律公共数据受控联网

- 状态：Accepted
- 日期：2026-07-14
- 决策级别：高风险信任边界

## 背景

最终产品既要处理合同、工资、考勤、聊天、身份证和银行账户等高度敏感案件材料，又必须持续发现法律、司法解释、北京地方规则、动态参数和提交要求变化。

完全离线会导致法律新鲜度无法证明；把全部案件数据放入托管服务会提前引入多用户隔离、身份、数据跨边界、备份、删除和事故响应责任。当前 v0.3 是本地单用户技术内核，也没有支撑托管真实案件的身份与隐私基础设施。

## 决策

第一阶段采用混合本地优先架构，并明确三个独立边界。

### 1. 案件数据面

- 原始材料、解析结果、事实、证据映射、计算和文书默认只保存在本地案件工作区；
- 原始文件只读，派生内容使用独立存储和内容摘要；
- 默认禁止案件数据面主动访问外部网络；
- 解析器不得执行宏、脚本、链接或文件中包含的指令；
- 引入真实案件前必须实现加密、脱敏、保留、删除和恢复控制。

### 2. 法律更新面

- 只允许访问经过版本化审核的官方来源注册表；
- 只获取公共法律、政策、官方案例和提交要求，不发送案件事实作为查询参数；
- 保存请求策略、原始响应字节、最终 URL、重定向链、响应元数据、内容哈希、抓取器版本和抓取时间；
- 对新旧内容生成差异和影响分析，但法律效力、版本关系和案件适用性仍需专业审核；
- 定期检查不能替代提交前强制新鲜度检查。

### 3. 外部模型面

- 默认关闭；
- 启用前必须明确供应商、数据保留、训练使用、地区、日志和删除政策；
- 只允许发送完成数据分类、最小化和用户确认的脱敏片段；
- 模型输出只作为提取或草拟候选，不能直接产生法律版本、专业金额、审批或提交状态。

## 状态与失败语义

提交候选材料生成前必须绑定一次成功的法律新鲜度检查。出现网络不可用、官方来源无法确认、抓取内容变化但未审核、规则审核过期或依赖版本冲突时：

1. 已有案件数据继续可读；
2. 旧规则快照不得被静默替换；
3. 受影响的结论、计算、文书和审批被标记失效或待复核；
4. 系统最多输出 `DRAFT` 或 `REVIEW_REQUIRED`，不得升级为 `SUBMISSION_CANDIDATE`。

## 未选择方案

### 完全离线

不选择。它可以降低网络泄露面，但无法在正式输出前证明法律、参数和提交要求仍然有效。

### 全量 SaaS

当前不选择。它有利于集中更新和多人协作，但在身份、租户隔离、数据保留、备份恢复和事故响应尚未建立时处理真实案件属于不可接受的风险扩大。

### 由大模型实时搜索法律

不选择。模型搜索不能替代原始响应冻结、版本关系、逐字引用、失败重放和专业法律审核。

## 影响

- 法律更新器成为明确的外部依赖，需要超时、大小、重定向、限速、缓存、重试和失败降级；
- 案件工作区与法律规则库使用摘要和版本引用连接，不共享隐式可变状态；
- 本地应用可以离线查看案件，但离线时不能产生提交候选材料；
- 将来引入 SaaS 时必须形成新的 ADR、威胁模型、数据处理协议和迁移方案，不能把本决策自动解释为授权上传案件数据。

## 重新评估条件

出现以下任一信号时重新评估：

- 多名真实用户必须同时编辑同一案件；
- 法律更新服务需要接收案件事实才能完成适用性判断；
- 官方来源要求登录、验证码、客户端证书或禁止自动获取；
- 产品进入托管生产环境或处理跨地区、跨组织真实案件；
- 外部模型供应商、数据保留或训练政策发生实质变化。

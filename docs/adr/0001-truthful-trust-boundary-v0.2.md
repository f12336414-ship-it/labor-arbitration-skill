# ADR-0001: v0.2 收缩为引用完整性内核

- 状态：Accepted
- 日期：2026-07-14
- 决策级别：高风险、可逆接口迁移

## 背景

v0.1 的确定性代码能验证结构、引用、快照和通用加法，但状态名、规则状态、证据状态和 JSON 审批可能被理解为法律正确或提交批准。项目没有权威法律内容、时效/金额引擎、证据语义、身份系统或真实案件验收。

## 决策

1. 产品显示名采用“劳动仲裁引用完整性内核”；仓库和 Skill 技术标识暂不改名，以避免安装路径和调用兼容性破坏。
2. 自动化上限改为 `REFERENCE_INTEGRITY_VALIDATED`，其后固定为外部 `PENDING_LEGAL_REVIEW`。
3. schema 1.1、`MACHINE_VALIDATED_CANDIDATE` 和 `HUMAN_APPROVED_FOR_SUBMISSION` 不提供向后运行兼容，只提供明确弃用错误。
4. 法律规则、时效、证据语义、举证责任、请求冲突、隐私审查、风险关闭和审批都保持未验证或外部待处理状态。
5. 官方域名白名单只作为候选过滤，禁止命名为法律来源验证。
6. 未通过法律/风险/身份/隐私设计门禁的生产能力建立阻断路线图，不以演示数据实现。

## 影响

- v0.2 是 pre-1.0 的有意破坏性安全迁移。
- 旧案件包必须按迁移指南修改并重新生成全部快照。
- 使用者获得更窄但可证明的保证；不能从本项目得到提交状态。
- 未来真实法律能力必须由有资质的领域负责人、风险负责人和可审计基础设施共同批准。

## 未选择方案

- 保留旧名称并只加免责声明：拒绝，因为状态字段可能脱离上下文传播。
- 用静态示例规则补齐时效/金额：拒绝，因为无法证明完整性、版本和适用性。
- 继续接受 `HUMAN` JSON 并要求使用者自律：拒绝，因为字符串不能认证身份或权限。
- 立即重命名仓库和 Skill：暂缓，因为技术标识是公开兼容接口；显示名已经收缩。

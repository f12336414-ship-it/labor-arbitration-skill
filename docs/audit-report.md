# 方案审核与整改记录

审核对象为“以可验证可靠性为目标的中国劳动仲裁 Skill 方案 V1.0”。结论是：原方案方向正确，但在开发前存在 8 项 P1 阻断问题；本仓库的 V1.1 契约和参考实现已对其中可由本地 Skill 解决的部分建立可执行门禁，托管与真实生产部署仍为 `NOT_READY`。

| 原始风险 | V1.1 处理 | 当前状态 |
| --- | --- | --- |
| 质量标准停留在描述，不能执行 | 稳定 finding code、退出码和 86 项行为测试 | 本地参考实现已覆盖 |
| 官方发布者与法律效力混同 | 分离 source artifact、document type、hierarchy、binding status | 已门禁 |
| 缺证即阻断与用人单位控制证据冲突 | 增加初步举证、控制方和调取请求模型 | 已门禁 |
| 时效仅用布尔值表示 | 结构化起算、中断、中止、期限与人工复核状态 | 已门禁 |
| 对抗复核和人工审批可被模型自授 | P0/P1 关闭、规则核验、隐私复核和批准均要求 HUMAN actor | 结构门禁已实现；生产身份认证待办 |
| 个人信息和外部模型边界不完整 | 默认本地单用户；外传条件和影响评估列为部署前决策 | 托管部署未就绪 |
| 提示注入与工具权限边界不清 | 输入始终是不可信数据；登记脚本无执行/网络能力 | 已控制并测试 |
| 候选结果缺少快照和失效生命周期 | 包、文档、依赖快照及 `REVALIDATION_REQUIRED` | 已建模 |

设计审核参考了以下官方来源，实际案件使用时仍必须按适用时间重新核验：

- [最高人民法院关于审理劳动争议案件适用法律问题的解释（二）](https://gongbao.court.gov.cn/Details/bb72019c45453f84d920bd6375573e.html)，2025-09-01 起施行；

- [中华人民共和国立法法](https://flk.npc.gov.cn/detail?id=ff808181865edc140186d9d6f1cb2503)；
- [中华人民共和国个人信息保护法](https://scca.miit.gov.cn/zwgk/zcwj/flfg/art/2024/art_1e503b8a7cbe4c8b8e973bb7c16cb56b.html)。

## 未关闭的生产阻断项

- 独立法律负责人、隐私负责人和风险接受人尚未确定；
- 完整北京规则包、动态更新责任和复核 SLA 尚未建立；
- 当前 JSON 审批记录没有独立身份认证或数字签名；
- 多租户授权、密钥、审计、留存、删除、备份和事件响应尚未设计；
- 尚无真实但合法脱敏的盲测集与独立法律验收。

因此，0.1.x 是公开可审计的本地参考实现，不是可直接承载真实案件的托管法律产品。

# 最终产品实施路线图

- 基线日期：2026-07-14

- 实时状态：[progress.md](progress.md)
- 需求基线：[final-product-requirements.md](final-product-requirements.md)

## 1. 状态规则

| 状态 | 含义 |
| --- | --- |
| `DONE` | 验收标准满足且已有代码、测试、审核或发布证据 |
| `IN_PROGRESS` | 已开始，尚未满足全部验收标准 |
| `FOUNDATION` | 已有可复用底座，但不能宣称目标能力实现 |
| `NOT_STARTED` | 尚未开始端到端实现 |
| `BLOCKED_EXTERNAL` | 技术工作可保留，但最终验收等待法律、安全或真实数据负责人 |

只有 `DONE` 计入完成任务数。路线图、Schema、示例或提示词本身不能把法律能力升级为 `DONE`。

## 2. 阶段依赖

```text
P0 产品与治理基线
 ├─> P1 证据工作区 ───────────────┐
 ├─> P2 法律规则系统 ──> P3 案件分析 ──> P4 计算系统
 └─> P6 安全治理基础 ─────────────┤
                                   └─> P5 交付系统 ──> P6 生产验收 ──> v1.0
                                                                          └─> P7 全国扩展
```

P1 和 P2 可以并行；P3 必须使用已版本化规则；P4 必须绑定已确认事实和规则；P5 不能绕过 P1–P4；P6 的隐私和安全控制必须在引入真实案件前到位。

## 11. 当前关键路径

当前优先顺序为：

1. P1-06/P1-07 可重放锚点、人工确认和事实状态模型；
2. P2-05/P2-09/P2-10 新鲜度调度、变化影响和更新监控；
3. P2-07/P2-08 北京规则包和用户交叉验证；
4. P3/P4/P5 依次形成第一条欠薪/工资差额端到端垂直切片；
5. P6 完成 PDF/OCR 独立沙箱、隐私安全、评测和真实案件生产门禁。

第一条垂直切片应选择一个高频且边界可控的请求，从原始证据、事实确认、规则版本、时效、金额、来源句到文书完整贯通，而不是先分别做完所有解析器或所有规则。
